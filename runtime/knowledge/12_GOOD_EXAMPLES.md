# 12 — Good Examples

Usa ejemplos buenos para few-shot. El runtime también carga ejemplos desde `examples/good`.

Patrones correctos:

- Incluyen todos los campos del pack.
- Usan coordenadas normalizadas.
- Separan OCR de interpretación.
- Marcan incertidumbres.
- Incluyen `DO_NOT_ASSUME`.
- Son útiles para acciones.

Ejemplos incluidos:

- `web_google_home_pack.md`
- `excel_sales_table_pack.md`
- `dashboard_kpi_pack.md`
- `gui_error_dialog_pack.md`
